# 만파식(MANPASIK) 브랜드 가이드라인

**문서번호**: MPK-BRAND-GUIDE-v2.0
**작성일**: 2026-02-14
**상태**: Official

---

## 1. 개요 (Overview)

본 문서는 **만파식(MANPASIK)** 브랜드의 등록 상표권을 보호하고, 모든 플랫폼(App, Web, OS)에서 일관된 브랜드 아이덴티티를 유지하기 위한 공식 가이드라인이다. 모든 개발자, 디자이너, 기획자는 본 가이드를 **절대 준수**해야 한다.

## 2. 공식 명칭 (Official Naming)

### 2.1 등록 상표 (Registered Trademarks)

| 언어 | 표기 | 사용 규칙 | 예시 |
| :--- | :--- | :--- | :--- |
| **국문** | **만파식** | 조사 없이 단독 사용 권장. 붙여쓰기 시 가독성 고려. | 만파식 생태계 (O), 만파식월드 (O) |
| **영문** | **MANPASIK** | **대문자 표기** 원칙. 로고 및 타이틀에 필수 사용. | MANPASIK World (O), MANPASIK (O) |

### 2.2 금지 및 주의 사항 (Don'ts)
- **만파식적**: 설화적 유래를 설명하는 역사적 맥락 외에는 브랜드명으로 사용 절대 금지.
- **Manpasik**: 일반 텍스트 문장 내 서술 외에, 브랜드 타이틀/로고로 소문자 혼용 금지.
- **만파**: 축약형 사용 지양.
- **MPK**: 내부 문서번호 접두어로만 사용. 사용자 노출 금지.
- **ManPaSik**: 카멜케이스 표기 금지 (코드 내부 변수명 제외).

## 7. 보이스 & 톤 (Voice & Tone)

### 7.1 브랜드 보이스
| 특성 | 설명 | 예시 |
|------|------|------|
| **신뢰** | 의료 데이터를 다루는 전문성 | "정확한 차동측정 기술로 건강을 관리합니다" |
| **따뜻함** | 사용자를 배려하는 친근함 | "오늘도 건강한 하루 되세요" |
| **명확함** | 쉽고 이해하기 쉬운 표현 | "혈당이 정상 범위입니다" (O) / "글루코스 수치가 euglycemic range" (X) |
| **격려** | 긍정적 동기 부여 | "꾸준한 측정이 건강의 시작입니다" |

### 7.2 맥락별 톤
| 맥락 | 톤 | 예시 |
|------|-----|------|
| 측정 결과 (정상) | 따뜻 + 격려 | "좋은 결과입니다! 계속 이 추세를 유지하세요." |
| 측정 결과 (주의) | 신뢰 + 명확 | "혈당이 높은 편입니다. 식단 조절을 권장합니다." |
| 측정 결과 (위험) | 명확 + 긴급 | "혈당이 위험 수준입니다. 의료 전문가와 상담하세요." |
| 에러/장애 | 명확 + 해결지향 | "연결이 끊어졌습니다. 다시 시도해주세요." |
| 마케팅 | 따뜻 + 격려 | "만파식과 함께 더 건강한 내일을 만들어보세요." |
| 의료 서비스 | 신뢰 + 전문 | "전문의가 측정 데이터를 기반으로 상담합니다." |

## 9. 모션 원칙 (Motion Principles)

### 9.1 기본 원칙
| 원칙 | 설명 |
|------|------|
| **의미 있는 전환** | 모든 애니메이션은 상태 변화를 전달 |
| **물의 흐름** | 만파식적 파도 모티프를 반영한 유기적 움직임 |
| **절제된 화려함** | 과도하지 않으면서 프리미엄 느낌 |

### 9.2 Duration 가이드

| 타입 | Duration | Easing | 용도 |
|------|----------|--------|------|
| Micro | 100-150ms | ease-out | 버튼 클릭, 토글 |
| Small | 200-300ms | ease-in-out | 카드 확장, 리스트 전환 |
| Medium | 300-500ms | cubic-bezier(0.4,0,0.2,1) | 페이지 전환, 모달 |
| Large | 500-800ms | cubic-bezier(0.0,0,0.2,1) | 화면 진입, 데이터 로딩 |
| Breathing | 3000ms | ease-in-out (loop) | 측정 중 호흡 애니메이션 |
| Wave | 2000ms | linear (loop) | 물결 배경 애니메이션 |

### 9.3 전용 애니메이션

| 이름 | 설명 | 트리거 |
|------|------|--------|
| Wave Ripple | 동심원 확산 파동 | 측정 버튼 탭 |
| Breathing Overlay | 호흡 스케일 펄스 | 측정 진행 중 |
| Sanggam Glow Line | 금색 그라데이션 라인 이동 | 컨테이너 강조 |
| Data Flow | 데이터 입자 흐름 | 데이터 동기화 |
| Success Check | 체크마크 드로잉 | 작업 완료 |

## 11. 적용 가이드 (Implementation)

### 11.1 UI/UX
- 앱 실행 화면(Splash), 로그인 화면, 헤더바에는 반드시 **'MANPASIK'** 대문자 로고를 사용한다.
- 한글 텍스트 내에서는 '만파식'을 사용하여 친숙함을 전달한다.

### 11.2 코드 (Code)
- 클래스, 변수명: `Manpasik` (CamelCase) 또는 `manpasik` (snake_case) 허용
- 사용자 노출 문자열: 위 네이밍 규칙 준수 필수
- 예: `class ManpasikApp extends StatelessWidget` (O) / `title: 'MANPASIK'` (O)

### 11.3 문서 (Documents)
- 공식 문서: "만파식" 또는 "MANPASIK" 사용
- 문서번호 접두어: "MPK-" (내부 사용만)
- 영문 참조 시: "ManPaSik" 사용 허용 (학술/기술 문서 한정)

---

**결론**: '만파식'과 'MANPASIK'은 단순한 이름이 아닌, 우리의 고유한 지적 재산이자 자부심이다. 이 명칭이 정확하게 사용될 때 브랜드의 가치는 극대화된다.
