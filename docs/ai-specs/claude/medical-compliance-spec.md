# 의료 규정 준수 분석 스펙 (Claude용)

## 프로젝트 정보

- **프로젝트명**: ManPaSik (만파식) Healthcare Ecosystem
- **의료기기 분류**: 체외진단의료기기 (IVD)
- **위험 등급**: Class II (중등도 위험)
- **대상 시장**: 한국, 미국, EU, 중국, 일본

---

## 분석 요청 사항

### 1. 규제 프레임워크 분석

| 국가 | 규제 기관 | 인증 경로 | 분석 필요 |
|------|-----------|-----------|-----------|
| 한국 | MFDS | 체외진단의료기기 허가 | ✓ |
| 미국 | FDA | 510(k) Pre-market Notification | ✓ |
| EU | Notified Body | CE-IVDR (EU 2017/746) | ✓ |
| 중국 | NMPA | 의료기기등록증 | ✓ |
| 일본 | PMDA | 체외진단용의약품 승인 | ✓ |

**산출물:**
- 각 국가별 인증 요구사항 체크리스트
- 기술문서(Technical File) 목차
- 임상시험 필요 여부 판단

---

### 2. 데이터 보호 규정 분석

| 규정 | 적용 지역 | 핵심 요구사항 |
|------|-----------|---------------|
| HIPAA | 미국 | PHI 보호, BAA 계약, 암호화 |
| GDPR | EU | 동의 관리, 삭제권, DPO |
| 개인정보보호법 (PIPA) | 한국 | 동의, 안전조치, 파기 |
| PIPL | 중국 | 현지화, 동의, 이전 제한 |
| APPI | 일본 | 동의, 안전관리조치 |

**분석 필요 항목:**
- 데이터 분류 (PII, PHI, 민감정보)
- 국가별 데이터 현지화 요구사항
- 동의 관리 UI 요구사항
- 데이터 보존 기간 및 삭제 정책

---

### 3. 품질경영시스템

**필수 표준:**
- ISO 13485:2016 (의료기기 품질경영시스템)
- IEC 62304:2006+AMD1:2015 (의료기기 소프트웨어 수명주기)
- ISO 14971:2019 (위험관리)
- IEC 62366-1:2015 (사용적합성)

**분석 필요 항목:**
- 소프트웨어 안전 등급 (Class A/B/C)
- 설계 및 개발 문서 요구사항
- 검증/확인 활동 정의
- SOUP(Software of Unknown Provenance) 관리

---

### 4. 사이버보안 요구사항

**참조 표준:**
- FDA Cybersecurity Guidance (2023)
- EU MDR Annex I, Section 17.4
- IEC 81001-5-1 (의료기기 사이버보안)

**분석 필요 항목:**
- 위협 모델링 (STRIDE)
- 취약점 관리 프로세스
- 소프트웨어 BOM (SBOM) 요구사항
- 침해사고 대응 계획

---

### 5. 시스템 특성 (분석용 컨텍스트)

**데이터 흐름:**
```
리더기 → BLE → 앱 → 클라우드 → DB
         ↓
      로컬 저장
```

**처리 데이터 유형:**
- 생체 측정 데이터 (혈당, 지질 등)
- 개인 식별 정보 (이름, 생년월일)
- 건강 기록 (FHIR R4 형식)
- 위치 정보 (선택적)
- 음식/운동 로그

**아키텍처 특성:**
- 오프라인 100% 동작 지원
- 엣지 AI 추론 (TFLite)
- 연합학습 (Federated Learning)
- E2E 암호화
